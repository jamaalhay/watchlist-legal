SMS Opt-In Process

How users consent to receive SMS messages from PikPal.

Phone Verification Flow

PikPal uses SMS solely for one-time verification codes during sign-in. No marketing, promotional, or recurring messages are sent.

How it works:

  1. The user navigates to the PikPal sign-in page.
  2. The user enters their phone number in the "Sign in with your phone" section.
  3. The user checks an unchecked-by-default consent checkbox that reads: "I agree to receive a one-time SMS verification code from PikPal to authenticate my login. Message frequency is limited to login requests. Message and data rates may apply. Reply STOP to opt out or HELP for assistance. Consent is not a condition of purchase. View our Privacy Policy and Terms of Service."
  4. The user taps "Send code" (button is disabled until the checkbox is checked) — this is the explicit opt-in action.
  5. A single 6-digit verification code is sent via SMS.
  6. The user enters the code to complete sign-in.

No SMS is sent without the user explicitly checking the consent checkbox and tapping "Send code." The checkbox is unchecked by default — consent is never forced or pre-selected. Users can also choose to sign in with Google instead, bypassing SMS entirely.

Message frequency & content:

  • Frequency: One message per sign-in attempt, only when the user requests it.
  • Content: A 6-digit numeric verification code. Example: "Your PikPal verification code is: 123456"
  • No recurring messages. No marketing. No promotions.

Opt-out:

Users can stop receiving SMS at any time by:

  • Replying STOP to any message to immediately opt out.
  • Replying HELP to any message for assistance.
  • Choosing to sign in with Google instead, bypassing SMS entirely.

Since messages are only sent on-demand when the user explicitly requests a login code, there is no subscription to cancel. Consent is not a condition of purchase or use of the app.
